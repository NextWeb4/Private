02关于

这里不是简介，是一份持续更新的工作记录。

我不是只想做一张漂亮名片。这个站点更像我的公开工作台：一边把交易复盘、阅读思考和日常判断写下来，一边把 Web 练习、自动化脚本和小工具做成可访问的项目。

NextWeb4 是我给自己的交付系统。博客留下判断形成的过程，GitHub 保存能运行的实现，网页把入口整理清楚；如果一个想法还不能被打开、被使用、被复盘，它就还没有真正交付。

所以这里的“关于”不是一句包装好的自我介绍，而是一组正在更新的证据：我关心什么问题，怎样拆解它，用什么方式做出来，又如何根据反馈继续修正。

  • 迭代

    保留可追踪的更新，把每次修正都变成下一次交付的基础。

04技术与方法

四条能力线，都有公开证据。

这些能力不是标签，而是博客文章、公开仓库和能运行的本地工具共同留下的证据。

  • 桌面产品与本地优先工具

    用 Python / PySide6、.NET / WinUI 和 Tauri，把局域网传输、翻译、号码与邮箱管理做成可安装、可备份、可审计的桌面产品。

    局域网文件传输↗ DeepSeek 翻译工作台↗ 号码绑定管理器↗
  • 配置、网络与离线工作流

    围绕 Clash / OpenClash / Mihomo YAML、字幕处理和文件夹保护，把高频本地操作整理成可检查、可导出、可回滚的离线流程。

    YAML 配置工作台↗ 离线工具套件↗
  • 检索、提示词与内容工程

    把 NFKC 归一化、中文二元分词与字段加权用于博客检索，同时把提示词和照片元数据整理成可搜索、可复用的本地内容资产。

    提示词管理器↗ 博客检索实现↗ 照片元数据编辑器↗
  • 权限、安全与可恢复交付

    把用户组权限、操作审计、AES-256-GCM 加密、SQLite 备份与发布校验放进交付链路，让工具不仅能运行，也能恢复和追责。

    权限与操作审计↗ 离线文件夹加密↗ 本地数据安全模型↗
